# 4. 코틀린 클래스와 객체

## 1. `data class`

데이터를 보유하는 목적으로 사용되는 클래스입니다. 컴파일러가 자동으로 다음 메서드들을 생성해줍니다.

- `equals()` / `hashCode()`
- `toString()`
- `componentN()` (구조 분해 선언 지원)
- `copy()`

```kotlin
data class User(val name: String, val age: Int)

val user1 = User("Alice", 25)
val user2 = user1.copy(age = 26) // 이름은 그대로, 나이만 변경하여 복사

val (name, age) = user1 // 구조 분해 선언
println("$name is $age years old")
```

## 2. `sealed class` / `sealed interface`

상속 계층을 제한하여, 컴파일러가 모든 하위 클래스를 알 수 있게 합니다. `when` 식에서 모든 경우를 처리했는지 검사할 수 있어 상태 관리에 매우 유용합니다.

```kotlin
sealed class UiState {
    object Loading : UiState()
    data class Success(val data: String) : UiState()
    data class Error(val exception: Throwable) : UiState()
}

fun handleState(state: UiState) {
    when (state) {
        is UiState.Loading -> showLoading()
        is UiState.Success -> showData(state.data)
        is UiState.Error -> showError(state.exception)
        // else 분기가 필요 없음 (모든 경우를 처리했으므로)
    }
}
```

## 3. `object`와 `companion object`

### object (싱글톤)

클래스 정의와 동시에 객체를 생성합니다. 전역적으로 유일한 인스턴스를 보장합니다.

```kotlin
object DataProvider {
    fun register() { /*...*/ }
}

DataProvider.register()
```

### companion object (동반 객체)

클래스 내부에 선언되는 객체로, 자바의 `static` 멤버와 유사한 역할을 합니다. 팩토리 메서드 패턴 구현에 자주 사용됩니다.

```kotlin
class MyClass {
    companion object Factory {
        fun create(): MyClass = MyClass()
    }
}

val instance = MyClass.create()
```

## 4. Enum vs Sealed 비교

- **Enum Class:** 상수 집합을 정의할 때 사용합니다. 각 상수는 단일 인스턴스입니다.
- **Sealed Class:** 상속 계층을 제한하지만, 각 하위 클래스는 여러 인스턴스를 가질 수 있으며 상태(데이터)를 포함할 수 있습니다.

```kotlin
enum class Color(val rgb: Int) {
    RED(0xFF0000),
    GREEN(0x00FF00),
    BLUE(0x0000FF)
}
```

## 5. 중첩 클래스 (Nested & Inner Class)

- **Nested Class (기본):** 외부 클래스의 멤버에 접근할 수 없습니다. (Java의 `static nested class`)
- **Inner Class (`inner`):** 외부 클래스의 참조를 가지며 멤버에 접근할 수 있습니다. (Java의 `inner class`)

```kotlin
class Outer {
    private val bar: Int = 1
    
    class Nested {
        fun foo() = 2
    }
    
    inner class Inner {
        fun foo() = bar // 외부 클래스 멤버 접근 가능
    }
}
```

## 6. 익명 클래스 (Anonymous Class)와 `object` 표현식

인터페이스나 추상 클래스를 즉석에서 구현하여 객체를 생성할 때 사용합니다.

```kotlin
window.addMouseListener(object : MouseAdapter() {
    override fun mouseClicked(e: MouseEvent) { /*...*/ }
    override fun mouseEntered(e: MouseEvent) { /*...*/ }
})
```

## 7. 함수의 유연성 (Default & Named Arguments)

코틀린은 자바의 메서드 오버로딩(Overloading)을 줄여주는 강력한 기능을 제공합니다.

- **Default Arguments (기본 인자):** 매개변수에 기본값을 지정하여, 호출 시 생략할 수 있습니다.
- **Named Arguments (지명 인자):** 매개변수의 이름을 명시하여 호출할 수 있어 가독성이 높아지고 순서에 구애받지 않습니다.

```kotlin
fun greet(name: String, message: String = "Hello") { ... }

greet("Alice") // message 생략 가능
greet(message = "Hi", name = "Bob") // 순서 변경 가능
```

## 8. 확장 함수 (Extension Function)

기존 클래스(String, View, List 등)를 수정하지 않고도 새로운 기능을 추가할 수 있는 방법입니다. **OCP(Open-Closed Principle)**를 준수하며 유틸리티 클래스를 대체하는 훌륭한 수단입니다.

- `this` 키워드(수신 객체)를 통해 해당 클래스의 인스턴스에 접근할 수 있습니다.
- 실제로는 정적 메서드(Static Method)로 컴파일되므로 실행 시점의 부하가 거의 없습니다.

```kotlin
fun String.lastChar(): Char = this.get(this.length - 1)

println("Kotlin".lastChar()) // 'n'
```
