# 3. 객체지향: 상속과 인터페이스

## 1. 클래스와 가시성 (Visibility Modifiers)

코틀린의 클래스는 기본적으로 `final`이며 `public`입니다. 상속을 허용하려면 `open` 키워드를 명시해야 합니다.

- **public:** 어디서든 접근 가능 (기본값)
- **private:** 같은 파일 또는 클래스 내부에서만 접근 가능
- **protected:** 하위 클래스에서만 접근 가능 (최상위 선언에는 사용 불가)
- **internal:** 같은 모듈(Module) 내에서만 접근 가능

### 멀티 모듈에서 중요한 `internal`

안드로이드 프로젝트를 여러 모듈로 나눌 때, `internal`은 해당 모듈 내부에서만 사용되는 구현 세부 사항을 숨기는 데 유용합니다. 외부 모듈에서는 접근할 수 없으므로 API 오염을 방지할 수 있습니다.

```kotlin
// Module A
internal class InternalHelper {
    fun doSomething() { ... }
}

// Module B
// val helper = InternalHelper() // 컴파일 에러! 접근 불가
```

## 2. 초기화(Initialization) 순서

객체 생성 시 초기화 블록(`init`)과 프로퍼티 초기화는 위에서 아래로 순차적으로 실행됩니다.

1. 주 생성자(Primary Constructor) 호출
2. 첫 번째 `init` 블록 및 프로퍼티 초기화
3. 두 번째 `init` 블록 및 프로퍼티 초기화 ...
4. 부 생성자(Secondary Constructor) 호출

```kotlin
class InitOrderDemo(name: String) {
    val firstProperty = "First property: $name".also(::println)
    
    init {
        println("First initializer block that prints $name")
    }
    
    val secondProperty = "Second property: ${name.length}".also(::println)
    
    init {
        println("Second initializer block that prints ${name.length}")
    }
}
```

## 3. 상속 (Inheritance)

코틀린의 모든 클래스는 `Any`를 상속받습니다. 사용자 정의 클래스를 상속받으려면 `open` 키워드가 필요합니다.

```kotlin
open class Base(p: Int)

class Derived(p: Int) : Base(p)
```

### 메서드 오버라이딩

메서드도 기본적으로 `final`입니다. 오버라이딩을 허용하려면 `open`을 붙여야 하며, 오버라이딩하는 쪽에서는 `override` 키워드를 사용해야 합니다.

```kotlin
open class Shape {
    open fun draw() { /*...*/ }
    fun fill() { /*...*/ } // 오버라이딩 불가
}

class Circle : Shape() {
    override fun draw() { /*...*/ }
}
```

## 4. 인터페이스 (Interface)

코틀린 인터페이스는 추상 메서드뿐만 아니라 구현된 메서드(Default Method)도 가질 수 있습니다. 상태(State)를 가질 수는 없지만 프로퍼티를 정의할 수는 있습니다.

```kotlin
interface MyInterface {
    val prop: Int // 추상 프로퍼티
    
    fun bar()
    fun foo() {
        // 본문이 있는 메서드 (선택적 오버라이딩)
        println("Default Implementation")
    }
}

class Child : MyInterface {
    override val prop: Int = 29
    
    override fun bar() {
        // 구현 필수
    }
}
```
