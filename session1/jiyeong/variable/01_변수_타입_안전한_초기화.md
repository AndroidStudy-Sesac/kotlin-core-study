# 1. 변수, 타입, 그리고 안전한 초기화

데이터를 어떻게 선언하고, 언제 초기화하며, Null을 어떻게 방어할 것인가

## 1. 변수와 프로퍼티 (val vs var)

코틀린에서는 변수를 선언할 때 `val` (읽기 전용)과 `var` (변경 가능) 키워드를 사용합니다. 하지만 코틀린의 변수는 단순한 변수가 아니라 **프로퍼티(Property)**입니다.

### 프로퍼티 (Property)와 캡슐화
프로퍼티는 **필드(데이터)**와 **접근자(Getter/Setter)**를 하나로 묶은 개념입니다.
- **캡슐화(Encapsulation):** 객체의 데이터를 외부에서 직접 접근하지 못하게 하고, 메서드(접근자)를 통해서만 조작하게 하여 데이터를 보호하는 객체 지향 원칙입니다.
- 코틀린은 변수 선언 시 자동으로 Getter/Setter를 생성하여 캡슐화를 지원합니다.

```kotlin
class Person {
    // 내부적으로 private field와 public getter/setter가 생성됨
    var name: String = "Jiyeong" 
}
```

### 백킹 필드 (Backing Field)
프로퍼티의 실제 값을 저장하는 숨겨진 변수입니다. 커스텀 접근자(Getter/Setter)를 만들 때 무한 재귀 호출을 방지하기 위해 `field` 키워드를 사용합니다.

```kotlin
class User {
    var name: String = "Unknown"
        set(value) {
            println("이름 변경: $value")
            field = value // this.name = value로 쓰면 무한 루프 발생!
        }
}
```

## 2. 코틀린의 타입 계층

- **Any:** 모든 클래스의 최상위 클래스 (Java의 Object에 해당)
- **Unit:** 반환 값이 없음을 의미 (Java의 void에 해당)
- **Nothing:** 함수가 결코 정상적으로 종료되지 않음을 의미합니다.
    - 예외를 던지거나(`throw`), 무한 루프를 돌 때 사용됩니다.
    - 모든 타입의 하위 타입(Bottom Type)으로 취급되어, 엘비스 연산자(`?:`) 등에서 유용하게 쓰입니다.

```kotlin
fun fail(message: String): Nothing {
    throw IllegalArgumentException(message)
}

// data가 null이면 fail()이 호출되고 프로그램 흐름이 종료됨 (타입 불일치 에러 없음)
val result = data ?: fail("Data is null") 
```

## 3. Null 안전성 (Null Safety)

코틀린은 컴파일 시점에 Null 가능성을 검사하여 `NullPointerException`을 방지합니다.

- **Nullable Type (`?`):** 타입 뒤에 `?`를 붙여야 null을 담을 수 있습니다.
- **Safe Call (`?.`):** null이 아닐 때만 실행하고, null이면 null을 반환합니다.
- **Elvis Operator (`?:`):** null일 경우 디폴트 값을 지정합니다.
- **Not-null Assertion (`!!`):** null이 아님을 강제로 단언합니다.

```kotlin
val text: String? = null
val length = text?.length ?: 0
```

## 4. 스마트 캐스트와 타입 검사

- **is / !is:** 타입 검사 연산자입니다.
- **Smart Cast:** 컴파일러가 타입을 검사한 후에는 자동으로 해당 타입으로 캐스팅해줍니다.
- **as? (Safe Cast):** 캐스팅에 실패하면 예외 대신 null을 반환합니다.

## 5. 참조 연산자 (::)

더블 콜론(`::`)은 **참조(Reference)**를 의미합니다. 함수, 클래스, 프로퍼티를 실행하지 않고 객체 자체로 참조할 때 사용합니다.

- **함수 참조:** `list.filter(::isEven)` (람다 대신 함수 자체를 넘김)
- **클래스 참조:** `String::class` (클래스 메타데이터 접근)
- **프로퍼티 참조:** `Person::age` (프로퍼티 정보 접근)

## 6. 지연 초기화 전략 (lateinit vs lazy)

### lateinit (var)
- `var` 프로퍼티에만 사용 가능합니다.
- 초기화되지 않은 상태에서 접근하면 예외가 발생합니다.
- `::variable.isInitialized`로 초기화 여부를 확인할 수 있습니다. (여기서도 참조 연산자 사용)

### lazy (val)
- `val` 프로퍼티에만 사용 가능합니다.
- 호출 시점에 초기화 블록이 실행되며, 그 결과가 저장됩니다. (첫 호출 시에만 연산 수행)

```kotlin
val heavyObject: String by lazy {
    println("초기화 수행!")
    "Heavy Result"
}
```
