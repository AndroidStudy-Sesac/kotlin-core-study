# 함수형 프로그래밍과 컬렉션

<details>
<summary><h2> 🛠️ 1. 함수형 프로그래밍과 확장 함수</h2></summary>
<br/>

### ① 고차 함수와 람다 (Higher-Order Functions & Lambdas)
- **고차 함수:** 함수를 매개변수로 받거나 반환하는 함수
- **람다 (Lambda):** 이름 없는 함수 (`{ x, y -> x + y }` 형태)
- **`it` 생략:** 매개변수가 딱 1개일 때는 `->`를 생략하고 암묵적 이름인 `it`으로 사용 가능
- **참조 연산자 (`::`):** 람다식에서 단순히 다른 함수를 호출할 때 코드 축약 (`list.forEach(::println)`)

> **💡 실무 팁:** 클릭 리스너나 콜백 처리 시, 자바처럼 무거운 인터페이스 대신 람다 하나로 간단하게 처리 가능

---

### ② 인라인 함수 (Inline Functions) — 성능 최적화
고차 함수 사용 시 발생하는 람다 객체 생성 비용을 줄이기 위한 키워드

- **`inline`:** 컴파일 시 함수 호출 위치에 코드 자체를 복사하여 람다 객체 생성 비용을 제거
- **`noinline`:** 특정 람다만 인라인 처리에서 제외하고 싶을 때 사용
- **`crossinline`:** 람다 안에서 `return`을 써서 함수 전체가 의도치 않게 종료되는 현상(Non-local return)을 원천 차단

---

### ③ 스코프 함수 (Scope Functions)
객체의 컨텍스트 내부에서 코드 블록을 실행하는 코틀린 관용구

| 함수 | 객체 참조 | 반환값 | 실무 팁 & 주요 용도 |
| --- | :---: | :---: | --- |
| **`let`** | `it` | 람다 결과 | `obj?.let { }` 형태로 Null 체크 후 안전하게 실행할 때 |
| **`run`** | `this` | 람다 결과 | 객체 초기화와 동시에 결과값을 계산해서 반환받을 때 |
| **`with`** | `this` | 람다 결과 | 생성된 객체의 속성을 연속해서 여러 번 쓸 때 (`with(binding)`) |
| **`apply`** | `this` | **객체 자신** | 객체 생성과 동시에 속성 초기화할 때 |
| **`also`** | `it` | **객체 자신** | 데이터 건드리지 않고 로그, 유효성 검사 등 부가 작업 시 |

---

### ④ 확장 함수 (Extension Functions)
수정할 수 없는 기존 클래스(`String`, `View` 등)에 함수를 추가
- **동작 원리:** 내부적으로는 객체를 첫 번째 인자로 받는 `static` 함수로 변환됨

> **🚨 주의 (오버라이드 불가)** 
>
> 정적 바인딩을 따르기 때문에 다형성(오버라이딩)이 적용되지 않음
> 
> 기존 멤버 메서드와 이름이 겹치면 무조건 멤버 메서드가 이김

<br/>

### ① 가변 vs 불변 컬렉션

| 타입 | 설명 |
| --- | --- |
| **`List` / `Set` / `Map`** | 읽기 전용 (안전성 보장) |
| **`MutableList` / `MutableSet` / `MutableMap`** | 수정 가능 (`add`, `remove` 등) |

> **💡 팁 (데이터 무결성 유지)**
>
> 클래스 내부에서는 `MutableList`로 다루고, 외부로 노출할 때는 `List`로 제한하는 **'Backing Property'** 패턴을 사용
>
> (`private val _items` -> `val items: List`)

---

### ② 컬렉션 연산 체이닝 (Pipeline)
`for`문 대신 함수 체인 방식을 사용하여 가독성을 높임

- **가공 연산:** `filter` (조건 필터링), `map` (변환), `flatMap` (평탄화), `fold` (누적 계산)
- **데이터 검사:** `any` (하나라도 맞는지), `all` (전부 맞는지), `none` (하나도 없는지)

> **💡 팁 (Sequence 지연 계산)**
>
> 데이터가 수만 개일 때 단순 체이닝을 쓰면 메모리가 터질 수 있음
>
> `.asSequence()`를 붙이면 모든 연산을 모아뒀다 한 번에 처리(지연 계산)하여 성능을 극대화함

---

### ③ 제네릭 (Generics)과 변성 (Variance)
타입 파라미터 `<T>` 로 타입 안정성 보장

- **불변성 (Invariant):** `Box<Int>`와 `Box<Number>`는 아무 관계가 없음
- **공변성 (`out` / Producer):** 읽기 전용(`val` 전용). 하위 타입을 상위 타입 자리에 사용 가능
- **반공변성 (`in` / Consumer):** 쓰기 전용(`var` 파라미터). 상위 타입을 하위 타입 자리에 사용 가능

> **💡 암기 팁:** **PECS** (Producer → Extends (`out`), Consumer → Super (`in`))
